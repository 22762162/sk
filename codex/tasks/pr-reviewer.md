# Codex 任务模板 · pr-reviewer（首轮机器评审）

> 使用方式：对 Claude Code 产出的每个 PR，以本模板驱动 Codex 做第一轮评审；人类工程师只看标注后的重点。

## 任务

评审 PR #{{PR_NUMBER}}（{{PR_TITLE}}），按下列清单逐项检查并输出标注报告。

## 检查清单

1. **边界条件**：判界比较的开闭区间是否与 spec 条款一致；±1 秒边界是否有测试；极端输入（极端年份、负 Unix 时间戳）是否处理。
2. **错误处理**：非法输入路径是否返回结构化错误而非 panic/未捕获异常；错误信息是否会泄露内部实现细节。
3. **时区陷阱**：是否有任何绕过注入数据、凭常量或系统时钟推断时区/夏令时的代码；本地时间与 UTC 的转换点是否唯一且显式。
4. **注入风险**：拼接进提示词/SQL/命令行的外部输入是否有转义或白名单；反序列化是否受 schema 约束。
5. **确定性**：新增代码是否引入网络、时钟、无种子随机（engine-paipan 内为零容忍）。
6. **纪律符合**：是否触碰 `rulebase/entries/`（应驳回）；用户可见文案是否含红线词；提示词改动是否附回归报告。

## 输出格式

按严重度分级的标注报告：
- `BLOCKER`：必须修复才能合入（附 文件:行 与修复建议）
- `WARN`：建议修复
- `NOTE`:供人类评审者关注的上下文
结尾给出单行结论：`建议合入 / 修复后合入 / 驳回`。
