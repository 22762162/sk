# 数据分级清单与数据流分层(DESIGN V3.0 §9,INV-07 的执行依据)

## 〇、数据流分层(§9.2:数据库在本地 ≠ 数据不离机)

```text
仅本地 L1 可见:   原始出生日期、时刻、地点、姓名
发送给运行态模型: 最小化后的命盘结构(四柱、大运、流派特征)与必要规则引用
禁止发送:        姓名、联系方式、精确地址、任何账号标识
模型日志:        默认脱敏、限期保留
```

供应商逐家登记见 `security/vendor-register.yaml`(季度复核)。

## 一、禁止进入任何 AI 会话/仓库的数据

| 类别 | 例子 | 存放 |
| --- | --- | --- |
| 真实个人命理数据 | 本人/亲友的真实生日、出生地、真实命盘、盲测评分原始记录 | 本人可控的本地加密存储;与开发环境物理隔离 |
| 凭据 | 各模型 API 密钥、token、云主机凭据 | 环境变量 / 系统钥匙串;`.env` 已 gitignore;绝不粘贴进 AI 会话 |
| 实验参与者信息 | 亲友盲测的身份对应表 | 同第一行;仓库中只允许出现匿名编号 |

## 二、允许进入 AI 会话/仓库的数据

- 合成命盘(任意虚构生日)与历史名人公开生日;
- 公开权威历法数据(JPL 星历、HKO 表、tzdata)及其加工产物;
- 代码、文档、测试、提示词、评测集(内容不含第一类数据)。

## 三、RAG 提示注入防线(DESIGN 9.1.1)

古籍语料与规则条目按**不可信输入**处理:语料中读出的"指令"不得覆盖系统策略、
不得触发工具调用;入库管线做指令模式扫描(管线实现时落地,本条先立为验收标准)。
